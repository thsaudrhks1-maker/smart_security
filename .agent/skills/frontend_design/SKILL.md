# 프론트엔드 디자인 표준 (Frontend Design)

## 1. 시인성 최우선 원칙 (Contrast Rule)

모든 텍스트는 밝은 배경(#FFFFFF 또는 #F8FAFC) 위에서 사용자가 즉각적으로 인지할 수 있을 만큼 선명해야 합니다.

- **메인 텍스트**: `#0f172a` (Deep Navy) - 제목, 주요 수치.
- **본문 및 레이블**: `#1e293b` (Dark Slate) - 폼 레이블, 일반 텍스트.
- **보조 정보**: `#475569` (Slate Gray) - 설명 문구, 메타 정보.
- **금기 사항**: 흰색 배경에서 흐릿한 회색(`#94a3b8` 이하)을 본문 글자로 사용하는 행위 절대 엄금.

## 2. 프리미엄 스크롤바 가이드 (Scrollbar Rule)

긴 사이드바나 리스트 영역에는 브라우저 기본 스크롤바를 숨기고 모던한 슬림 스크롤바를 사용합니다.

- **규격**: 두께 `6px`, 모서리 곡률 `10px`.
- **배경**: `transparent` (투명).
- **핸들(Thumb)**: `rgba(148, 163, 184, 0.3)` (반투명 슬레이트).
- **인터랙션**: 호버 시 선명도 증가 (`0.5`).

## 3. 디바이스별 레이아웃 패턴 (Adaptive UI)

- **데스크탑 (Admin/Manager)**: 
    - 사이드바 고정형 레이아웃 (`MainLayout.jsx`).
    - 정보 밀도가 높은 데이터 그리드 중심.
- **모바일 (Worker)**:
    - 바텀 내비게이션 기반 UI (`MobileLayout.jsx`).
    - 큰 버튼과 터치 친화적 카드 레이아웃.
    - 현장에서의 시인성을 위해 고대비 디자인 적용.
