문서 관리자 스킬 (Documentation Manager Skill)

이 스킬은 에이전트가 'Project_Docs' 폴더를 지능적으로 관리하도록 합니다. 
경고: 사용자가 명시적으로 "문서 작성해", "로그 남겨"라고 명령하지 않는 한, 스스로 문서를 업데이트하지 마십시오. 매번 문서를 작성하면 속도가 느려집니다.
사용자가 명령을 내리면 아래 절차를 엄격히 따르십시오.

1. 디렉토리 구조 규칙
- 모든 문서는 다음 경로에 저장: [WorkspaceRoot]/Project_Docs/
- 데일리 로그 파일명: YYYY-MM-DD.txt (예: 2026-01-29.txt)

2. 내용 구성 및 포맷 규칙 (중요)
필수: 모든 내용은 반드시 '한글'로 작성해야 합니다. 사용자가 영어를 이해하기 어려워하므로, 기술 용어를 제외한 모든 설명은 한국어로 풀어 쓰십시오.
주의: 파일 내용 내부에는 마크다운 문법(#, **, [] 등)을 절대 사용하지 마십시오. 오직 일반 텍스트(Plain Text)와 구분선만 사용하십시오.

섹션 1: Daily Log (업무 일지)
- 사용자 요청과 에이전트 실행 내역을 요약합니다.
- 포맷:
  [시간] 업무 명
  - 세부 사항 1
  - 세부 사항 2

섹션 2: Process (진행 상황 및 의사결정)
- 현재 개발 단계(예: 기획, 프로토타입, 리팩토링)를 설명합니다.
- 중요한 기술적 의사결정(예: "라이브러리 X 대신 Y 사용 - 이유 Z")을 기록합니다.

섹션 3: Skill & Tech Stack (사용 기술 스택)
- 새로 추가되거나 현재 활발히 사용 중인 기술만 나열합니다.
- 가능한 경우 버전을 명시합니다.
- 포맷: 분류: 도구/라이브러리

3. 실행 템플릿 (작성 예시)
데일리 로그를 업데이트할 때는 기존 파일 뒤에 내용을 추가(APPEND)하거나, 파일이 없으면 새로 생성하십시오.

=== [시간] 업데이트 ===

[Daily Log]
1. 작업 요약
   - 상세 내용 A
   - 상세 내용 B

[Process]
- 현재 단계: ...
- 핵심 의사결정: ...

[Tech Stack]
- 스킬: ...
- 라이브러리: ...

4. 실행 트리거
사용자가 다음과 같이 말할 때 이 스킬을 활성화하십시오:
- "문서 작성해"
- "로그 남겨"
- "기록해"
- "Project_Docs 정리해"
