# Demo Seeding Skill

데모 및 프리젠테이션을 위한 풍부한 현장 데이터를 생성하고 관리하는 규칙입니다.

## 1. 개요
상무님 등 주요 의사결정자에게 보여줄 "살아있는 시스템"을 만들기 위해, 실제 건설 현장과 유사한 데이터를 자동으로 생성합니다.

## 2. 데이터 생성 원칙
- **실명 기반:** '테스트1', 'UserA' 대신 '김철수 소장', '이한성 대리' 등 실제 사람 이름을 사용합니다.
- **현장감 있는 내용:** 작업 설명에 '작업내용' 대신 '104동 지하 2층 전기 배선 및 분전함 설치'와 같은 구체적인 텍스트를 사용합니다.
- **날짜 동기화:** 작업 계획 및 출근 기록은 항상 **실행 시점의 오늘 날짜**를 포함하도록 생성합니다.
- **위험 요소 매칭:** 고소 작업이 있으면 반드시 '추락 위험'이라는 위험 구역 정보가 함께 생성되어야 합니다.

## 3. 폴더 구조
- `back/scripts/seeding/`: 데이터 생성 스크립트 보관
- `back/scripts/seeding/master_data.py`: 업체, 템플릿 등 기본 데이터 정의
- `back/scripts/seeding/scenario_seeder.py`: 특정 시나리오(예: 평화로운 오전 현장, 사고 발생 상황)를 시뮬레이션하는 스크립트

## 4. 실행 절차
1. 가상환경 활성화
2. `python -m back.scripts.seeding.scenario_seeder` 실행
3. 프론트엔드 새로고침하여 데이터 확인

## 5. 필수 포함 시나리오
- **신규 프로젝트 개설 시나리오:** 시공사, 협력사, 소장/안전관리자 자동 배정
- **일일 작업 할당 시나리오:** 작업자별 2~3개의 작업 할당 및 출근 처리
- **긴급 상황 시범 시나리오:** 위험 구역 감지 및 작업자 알림 전송 로직 포함 데이터
