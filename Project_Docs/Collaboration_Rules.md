# 스마트 시큐리티 협업 규칙 (Collaboration Rules)

## 1. 명령어 실행 권한 (Execution Policy)
- **원칙**: 에이전트는 사용자의 명시적 지시가 없는 한, **서버 접속, Docker 제어, DB 조작, 배포** 등 주요 시스템 변경 명령을 **직접 실행하지 않는다.**
- **역할**: 에이전트는 **"실행할 명령어"**와 **"예상 결과"**를 제공하고, 사용자가 이를 직접 터미널에 입력하여 학습하고 검증하도록 유도한다.

## 2. 작업 절차 (Workflow)
1.  **설명 (Explain)**: 현재 상황과 수행해야 할 작업의 목적 설명.
2.  **명령어 (Command)**: 사용자가 실행할 명령어(Snippet) 제공.
3.  **확인 (Verify)**: 사용자가 실행 결과를 공유하면, 에이전트가 이를 분석하여 다음 단계 제안.

## 3. 예외 (Exceptions)
- 단순 파일 읽기(`cat`, `ls`), 로컬 파일 생성/수정은 에이전트가 수행 가능.
- 사용자가 "이건 네가 해"라고 명시적으로 위임한 경우에만 직접 실행.
