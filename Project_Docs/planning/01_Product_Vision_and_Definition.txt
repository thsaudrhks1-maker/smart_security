==============================================================
PMIS-X Safe-On Lite 기획서 (정리본)
==============================================================

"2013년의 정통성을 2026년의 기술로 재해석하다."
소규모 현장을 위한 가볍고(Lite), 빠르고(Instant), 똑똑한(Context-aware) 안전 운영체계

--------------------------------------------------------------
1. Product Vision (2013 -> 2026)
--------------------------------------------------------------

1.1 정통성 (Heritage)
우리는 2013년 '스마트안전지킴이'를 통해 이미 작업자 위치, 출역, 위험구간 관리를 시도했습니다.
그때는 시대를 너무 앞서갔지만, 지금은 기술(AI, Digital Twin, Mobile)이 우리의 비전을 실현할 수 있습니다.
Safe-On Lite는 "우리가 원래 하던 일을 이제 제대로 완성하는 것"입니다.

1.2 핵심 철학의 진화
(구분)        (2013 스마트안전지킴이)           (2026 PMIS-X Safe-On)
[관점]        사람(Worker) 중심                상황(Context) 중심
              (누가 어디 있나?)                 (누가/어떤 공정/어떤 위험 상태인가?)
              
[방식]        점검/기록 (체크리스트)            예측/판단 (Risk Score 선제 경고)

[교육]        정기 집체 교육                   Just-in-Time (작업 투입 직전 30초)

[역할]        관리자의 경험에 의존              AI Co-Pilot (시스템 경고 + 관리자 판단)

[가치]        안전 비용 (Cost)                 수주/평가 자산 (Asset)

--------------------------------------------------------------
2. Safe-On Lite 제품 정의 (Target: 소규모 현장)
--------------------------------------------------------------

"BIM 없이, Primavera 없이, 즉시 도입 가능한 Safety Operating Layer"

소규모 현장(리모델링, 근생 등)은 복잡한 시스템을 구축할 여력도, 시간도 없습니다.
Safe-On Lite는 "오늘의 작업(Daily Work Plan)" 하나만 입력하면 안전 관리가 자동으로 굴러가는 구조입니다.

(핵심 가치)
1. 구축 Zero: 템플릿에서 '오늘의 작업'만 선택하면 끝.
2. 자동 경고: 작업+장소+날씨를 분석해 Risk Score와 필수 안전조치를 자동 제시.
3. 자동 증빙: 1분 점검과 조치 사항이 자동 로그로 남아 법적 대응력 확보.

3. 6대 핵심 기능 (MVP Requirement)
--------------------------------------------------------------

(F1) 출역/인력 (Attendance)
  - QR/간편번호로 10초 만에 출역 등록.
  - 상태 관리: On-Site(현장 내) / Off-Site / Restricted(출입제한)

(F2) 오늘의 작업 (Daily Work Plan) *핵심*
  - 복잡한 공정표(WBS) 대신 "작업 템플릿" 선택 (예: 2층 A존 / 거푸집 작업 / 크레인 사용).
  - 템플릿 선택 시 기본 위험요인, 필수 보호구, 점검 항목이 자동 로딩됨.

(F3) 작업구역 (Zone)
  - 도면/BIM 없이 직관적 설정: Level(층) + Zone(구역).
  - 모든 작업은 Zone에 귀속됨.

(F4) 위험 경고 엔진 (Risk Engine)
  - Rule 기반 자동 계산: 작업유형 + Zone + 장비 + 환경(시간/날씨) = Risk Score
  - Action: High Risk(70점 이상) 시 즉시 경고 팝업 및 필수 점검 강제.

(F5) 1분 점검 (Smart Inspection)
  - Pre-Start: 작업 전 1회, 자동 생성된 체크리스트 확인.
  - Evidence: 사진 1장 + 확인 버튼으로 끝. (복잡한 서류작업 제거)

(F6) 마이크로 교육 (Micro Training)
  - 작업 시작 버튼 누르면 30초 안전 브리핑 자동 노출.
  - 교육 이수 로그 자동 저장.

--------------------------------------------------------------
4. UX 시나리오 (현장 흐름)
--------------------------------------------------------------

1. 출근 (07:00): 작업자 앱 켜고 QR 찍고 출근.
2. 작업 확인 (07:10): "오늘 내가 할 일은 2층 거푸집 작업이구나." (앱 확인)
3. TBM/교육 (07:20): 앱에서 30초 안전 수칙 확인 ("오늘 크레인 들어오니 상부 주의하세요").
4. 작업 시작 (08:00): 작업 시작 버튼 -> 위험 경고 확인 ("현재 강풍 주의!").
5. 점검 (08:05): 안전띠 착용 사진 1장 업로드 OK.
6. 작업 중: 위험 상황 발생 시 "원터치 신고".
7. 퇴근: 작업 종료 및 퇴근 처리.

--------------------------------------------------------------
5. 데이터 모델 & 알고리즘 (Draft)
--------------------------------------------------------------

(Risk Score Algorithm - v1.0 Rule-Based)
  Risk Score = (WorkType 기본점수) 
             + (Zone 가중치: 옥상/PIT 등) 
             + (Equipment 가중치: 크레인/용접 등) 
             + (환경 가중치: 야간/우천/강풍)

  * 0~39 (Low): 일반 관리
  * 40~69 (Medium): 주의 (노란색)
  * 70~100 (High): 경고 (빨간색) + 작업 중지 권고 / 필수 점검 강제

(주요 엔티티)
  - User (Worker/Manager)
  - Site / Zone
  - WorkTemplate (라이브러리)
  - DailyWorkPlan (실제 작업 인스턴스)
  - SafetyLog (점검/사고 기록)
