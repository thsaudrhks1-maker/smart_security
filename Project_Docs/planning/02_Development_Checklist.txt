Phase 1: 기초 공사 (Foundation) - 완료
--------------------------------------------------------------
시스템의 뼈대를 만드는 단계입니다.

[v] 프로젝트 초기 세팅 (FastAPI, React, Vite)
[v] DB 스키마 설계 (1차)
    - User (사용자)
    - Company/Site (현장)
    - Zone (구역 - 기초)
[v] 인증 시스템 (Auth)
    - 로그인 (JWT) / 토큰 파싱 로직 개선 (Multiple Underscores 대응)
    - 역할별 자동 리다이렉트 (Admin/Manager/Worker)
    - 마스터키(0000) 로그인 기능 (개발용)
    - CORS 로컬 환경 최적화
[v] 공통 유틸리티
    - 하이브리드 DB 패턴 (ORM + Raw SQL)
    - 날짜/시간 데이터 표준화 (Object-based Date Handling)
    - 표준 예외 처리 및 대시보드 방어 코드 (Safe Call)

--------------------------------------------------------------
Phase 2: 핵심 데이터 흐름 (Core Data Flow) - [Focus Today]
--------------------------------------------------------------
사용자 요청: 관리자가 입력한 콘텐츠(작업/안전)를 작업자가 실시간으로 확인하고 크로스체크하는 구조 완성.

(2.1 작업 기준 정보 - Filling Contents)
[ ] 작업 템플릿(Work Template) 시딩
    - 공종별 표준 작업 데이터 구축 (예: 철근, 거푸집, 비계, 용접 등)
    - 공종별 필수 안전 수칙 및 위험 요인 매핑
[ ] 구역(Zone) 기초 데이터 구축
    - 현장 도면 기준 구역 정의 (예: 101동 1층, 2층, 지하주차장 등)

(2.2 일일 작업 계획 (Daily Work Plan) - Manager View)
[ ] 작업 계획 수립 API
    - 날짜 + 구역 + 작업 템플릿 선택 = "오늘의 작업" 생성
    - 작업 반장/팀원 할당 기능
[ ] 작업 계획 관리 UI (ManagerDashboard/DailyPlan)
    - 일일 작업 현황판 (칸반 보드 또는 리스트)
    - 위험도(Risk) 자동/수동 평가 입력

(2.3 위험 정보 및 안전 조치 (Risk & Safety))
[ ] 위험성 평가 연동
    - 작업 생성 시 관련된 위험 요인 자동 로딩
    - 관리자가 추가 안전 지시 사항 입력 (예: "강풍 주의, 상부 작업 금지")

Phase 3: 현장 운영 및 공유 (Worker View & Cross-Check)
--------------------------------------------------------------
관리자가 입력한 데이터를 작업자가 모바일/태블릿에서 확인합니다.

(3.1 작업자 전용 뷰)
[ ] "오늘의 내 작업" 확인 (My Work)

[ ] 안전 정보 열람
    - 관리자가 남긴 위혐 요인 및 안전 수칙 확인
[ ] 작업 시작/종료 보고
    - 현장 도착 확인 및 작업 시작 버튼 구현

(3.2 점검 & 로그 - Inspection & Log)

[ ] 안전 로그 (Safety Log)
    - 사고/아차사고 신고 기능
    - 로그 타임라인 조회 UI
