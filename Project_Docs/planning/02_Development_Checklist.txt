==============================================================
개발 체크리스트 (Dev Checklist)
==============================================================

목표: Safe-On Lite MVP (최소 기능 제품) 완성
이 문서를 매일 업데이트하여 진행 상황을 추적합니다.

--------------------------------------------------------------
Phase 1: 기초 공사 (Foundation) - 완료
--------------------------------------------------------------
시스템의 뼈대를 만드는 단계입니다.

[v] 프로젝트 초기 세팅 (FastAPI, React, Vite)
[v] DB 스키마 설계 (1차)
    - User (사용자)
    - Company/Site (현장)
    - Zone (구역 - 기초)
[v] 인증 시스템 (Auth)
    - 로그인 (JWT)
    - 회원가입 (Worker/Manager)
    - 비밀번호 암호화 (bcrypt)
    - API 구조 리팩토링 (api/authApi.js 등)
[v] 공통 유틸리티
    - 하이브리드 DB 패턴 (ORM + Raw SQL)
    - 표준 예외 처리 (Exception Handlers)
    - 프론트엔드 API 클라이언트 (client.js)

--------------------------------------------------------------
Phase 2: 핵심 로직 (Core Business Logic) - 진행 중
--------------------------------------------------------------
Lite 버전의 심장인 "오늘의 작업"과 "위험 계산"을 구현합니다.

(2.1 작업 템플릿 & 계획)
[ ] Work Template 라이브러리 구축 (DB Seed)
    - 공종별 기본 위험도, 필수 장비, 가중치 데이터 정의
[ ] Daily Work Plan API (/work/plans)
    - 작업 생성 (Template + Zone + 날짜 선택)
    - 작업 목록 조회 (날짜별/구역별)
    - 작업 상태 변경 (예정 -> 진행 -> 완료)
[ ] Work Plan UI (WorkList.jsx)
    - 작업 등록 모달/폼 구현
    - 작업 카드 디자인 고도화 (위험도 색상 표시)

(2.2 위험 경고 엔진 - Risk Engine)
[ ] Risk Score 계산 로직 구현 (백엔드)
    - calculate_risk(template, zone, equipment, env) 함수 구현
[ ] 위험 경고 UI
    - High Risk 작업 시 시각적 알림 (배지, 테두리 등)
    - Dashboard에 '오늘의 고위험 작업' 요약 위젯

(2.3 구역 관리 - Zone)
[ ] Zone 관리 API: 층(Level) / 구역(Area) 등록
[ ] Zone 선택 UI: 작업 등록 시 Dropdown 또는 간편 선택기

Phase 3: 현장 운영 (Site Operation)
--------------------------------------------------------------
작업자가 실제 현장에서 쓰는 기능들입니다.

(3.1 출역 & 작업자 관리)

[ ] 출역 체크 API (QR 또는 간편 버튼)
[ ] 작업자 배정 (Work Plan에 작업자 매핑)

(3.2 점검 & 로그 - Inspection & Log)
[ ] 1분 점검 기능

[ ] 안전 로그 (Safety Log)
    - 사고/아차사고 신고 기능
    - 로그 타임라인 조회 UI
